# 2026-07-20 工作日志

## [11:56] 「一键粘贴链接」扩展为「链接 / 上传图片」双模式

**用户需求**：把"一键粘贴链接"功能扩展为既可以粘贴链接，也可以直接上传服装图片。

**改动**：
1. **`app/src/components/LinkInput.tsx`**（核心重写）：
   - 顶部新增模式切换分段控件：`粘贴链接` / `上传图片`。
   - 链接模式：保持原有输入 + "识别服装" + "示例商品"。
   - 图片模式：隐藏 `<input type=file accept=image/*>` + 虚线拖拽区按钮；选图后 `FileReader` 读为 dataURL → `compressDataUrl(raw,1280,0.85)` 压缩 → 构造 `ParsedProduct`（imageUrl=压缩后 dataURL、isClothing=true、garments=[]、incomplete=true、platform="unknown"、title=文件名）→ `onParsed`。
   - 复用 `ProductCard` 已有的 `needManual`（garments 为空）分支：上传后自动进入"手动选择服装类型/颜色"流程，用户确认即试衣。
2. **`app/src/components/StepIndicator.tsx`**：步骤名 `粘贴链接` → `添加商品`。
3. **`app/src/App.tsx`**：FEATURES[2] desc 改为"粘贴购物平台链接，或直接上传服装图片"；步骤卡片标题改为"添加商品（链接 / 图片）"。

**关键链路验证**：
- 后端 `server/tryon.js` 的 `resolveToDataUrlCached` 对 `data:` 前缀直接复用（`src.startsWith("data:") return src`），故上传图片的 dataURL 作为 `product.imageUrl` 传给后端可直接当商品图使用（真试衣路径正确）。
- 本地 Canvas 预览（`tryOn`）仍用占位 SVG（`garmentDataUrl`），不使用上传图——与链接解析商品一致的限制，仅真 AI 后端路径用真实图。
- `tsc -b` 与 `vite build` 均通过（产物 `server/public/`）。

**注意**：直接上传未做 AI 自动识别服装类型，需用户在"手动选择"里指定（与平台反爬拦截时的 incomplete 流程一致）。若后续要做"上传即自动识别类型"，需新增后端视觉识别端点（当前无此前端→后端接口）。
