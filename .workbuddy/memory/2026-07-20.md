- **`app/src/components/LinkInput.tsx`**：`handleImage` 改用 `isImageFile` 判定；读取改用 `readImageFileAsDataUrl`；`<input accept>` 改为 `image/jpeg,image/png,image/webp`；提示文案加 WEBP。

## [18:11] 修复 api/parse 返回的 garments.detail 为空字符串

**用户反馈**：通过商品链接（`/api/parse`）识别的商品，返回 `garments.detail` 是空字符串，不正确，要求补全。

**根因**：上一轮只在「视觉模型成功返回 detail」时才填 detail。但生产环境里 detail 经常为空——① 淘宝主图有防盗链，服务端下载被拦/图过大退回 URL，视觉模型读不到图 → 返回空或 null；② 模型本身未返回 detail 字段；③ `meta.hasImage` 为 false 时 `canVision=false` 整段跳过。这些情况下普通购物链接分支（标题命中关键词）创建的 garment 完全没有 detail 字段 → 空字符串。

**改动（`server/parse.js`，纯后端）**：
- 新增 `buildGarmentDetail(title, region, price)`：从 HTML 元数据兜底生成非零 detail——按标题关键词抽取主色/图案/面料/版型（中文词表匹配），拼上试衣部位与参考价，如「商品为上装，主色白，图案为纯色，面料棉，版型宽松，参考价约¥129，（可补充更细致的视觉描述）」。
- 正常解析路径：标题命中关键词建 garment 时直接带 `buildGarmentDetail(...)`。
- `canVision` 视觉增强分支：`detail = rec.detail.trim() || buildGarmentDetail(...)`（视觉优先，空则回退）。
- `tryVision` 兜底分支：同样 `rec.detail.trim() || buildGarmentDetail(...)`。

**验证**：单测（mock fetch，VISION_API_KEY=test-key）覆盖三态——视觉返回「丰富 detail」→ 用之；「空 detail」→ 回退 HTML 兜底；「视觉调用失败(null)」→ 回退 HTML 兜底。三种情况 garments[0].detail 均 NON-EMPTY。两个 server 文件 `node --check` 通过。

**结论**：现无论视觉模型是否配置/成功，链接识别的 garment 都带非零 detail，前端「服装描述」输入框必有内容、试衣 prompt 也有参考。仍需**重启/重部署**后端生效。
