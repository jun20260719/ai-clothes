## [13:30] 上传图片支持 WEBP 格式识别

**用户需求**：上传的图片（商品图 / 自拍）要能识别并正确处理 WEBP 类型。

**根因**：① 部分下载的 `.webp`（淘宝/微信保存）`file.type` 为空或 `application/octet-stream`，被 `file.type.startsWith("image/")` 误判为非图片而拒绝；② `FileReader.readAsDataURL` 会沿用该错误 MIME 写进 dataURL 前缀，导致后续 `<img>`/canvas 无法解码（报"图片加载失败"）。

**改动**：
- **`app/src/lib/image.ts`**（新增 3 个导出）：`IMAGE_EXT`（含 webp/heic 等）、`EXT_MIME`（扩展名→MIME）、`isImageFile(file)`（MIME 优先，缺失按扩展名兜底）、`readImageFileAsDataUrl(file)`（读为 dataURL 并按扩展名修正 MIME，如 webp→`data:image/webp;base64,...`，确保可被 `<img>` 解码）。
- **`app/src/components/LinkInput.tsx`**：`handleImage` 改用 `isImageFile` 判定；读取改用 `readImageFileAsDataUrl`；`<input accept>` 改为 `image/jpeg,image/png,image/webp`；提示文案加 WEBP。
- **`app/src/components/SelfieUpload.tsx`**：`handleFile` 改用 `isImageFile(file) && !isHeic(file)` 判定，非 HEIC 分支读取改用 `readImageFileAsDataUrl`；`<input accept>` 加 webp；提示文案加 WEBP。

**验证**：`tsc -b` + `vite build` 均通过（产物 `server/public/`）。前端统一 `compressDataUrl(→JPEG 1280/0.85)` 兜底，webp 修正 MIME 后可正常解码、压缩并送视觉识别；后端仅收 JPEG，无需改动。

- **前端 `api.ts`**：`TryOnApiPayload.garment` 增加 `detail?: string`。
- **前端 `tryon.ts`**：`generateTryOn` payload 透传 `garment.detail`。
- **前端 `ProductCard.tsx`**：试衣部位下方新增「服装描述」Textarea，绑定 `selectedGarment.detail`，`onDetailChange` 回传。
